# Analisis WebGIS Bencana di Indonesia (Sudut Pandang QA Tester)

## 1. WebGIS BNBP – Dashboard Penanganan Darurat Banjir dan Longsor Provinsi Aceh, Sumatera Utara, dan Sumatera Barat Tahun 2025
https://gis.bnpb.go.id/bansorsumatera2025/

### Kelebihan

- **Data real-time dan terverifikasi**  
  Data diperbarui berdasarkan laporan lapangan yang telah melalui proses validasi sehingga dapat dipercaya sebagai sumber informasi resmi.

- **Integrasi data spasial dan statistik**  
  Sistem tidak hanya menampilkan peta, tetapi juga grafik dan rekap angka korban maupun kerusakan infrastruktur sehingga memudahkan analisis dampak bencana.

- **Mendukung pengambilan keputusan**  
  Informasi seperti jumlah korban, lokasi terdampak, dan fasilitas rusak membantu pemerintah dalam menentukan prioritas penanganan.

- **Transparansi informasi publik**  
  Data dapat diakses masyarakat secara terbuka sehingga meningkatkan transparansi penanganan bencana.

### Kekurangan

- **Navigasi peta kurang intuitif**  
  Beberapa dashboard WebGIS pemerintah sering memiliki struktur menu yang tidak langsung dipahami oleh pengguna awam.

- **Keterbatasan fitur analisis pengguna**  
  Pengguna hanya dapat melihat data, tetapi tidak dapat melakukan analisis lanjutan seperti filter waktu atau perbandingan data historis secara fleksibel.

- **Kurangnya integrasi data lingkungan real-time**  
  Informasi seperti curah hujan, tinggi muka air, atau sensor lapangan belum selalu terintegrasi langsung.

- **Akses mobile belum optimal**  
  Beberapa WebGIS pemerintah masih kurang responsif di perangkat mobile.

### Saran

- Integrasi data sensor real-time (curah hujan, tinggi sungai, cuaca).
- Early warning system berbasis peta yang memberikan peringatan otomatis.
- Mode analisis waktu (time slider) untuk melihat perkembangan bencana dari waktu ke waktu.
- Integrasi data pengungsian dan jalur evakuasi.

## 2. WebGIS Wilayah Terdampak Banjir Kota Tangerang
https://maps.tangerangkota.go.id/wilayahterdampakbanjir/

### Kelebihan

- **Menyediakan berbagai dataset geospasial**  
  Portal memuat banyak layer seperti batas wilayah, hidrografi, transportasi, utilitas, dan kebencanaan.

- **Mendukung kebijakan Satu Peta**  
  Integrasi data dari berbagai dinas memungkinkan konsistensi data spasial pemerintah daerah.

- **Mudah diakses publik**  
  Sistem berbasis web sehingga dapat digunakan oleh masyarakat tanpa instalasi aplikasi.

- **Potensi integrasi lintas sektor**  
  Data geospasial dapat digunakan untuk perencanaan kota, mitigasi bencana, dan pengembangan wilayah.

### Kekurangan

- **Fokus informasi kebencanaan masih terbatas**  
  Sistem lebih berfungsi sebagai geoportal data dibandingkan sistem monitoring bencana.

- **Visualisasi dampak banjir kurang detail**  
  Tidak semua data menampilkan informasi seperti kedalaman banjir, waktu kejadian, atau area evakuasi.

- **Kurangnya fitur analitik atau prediksi**  
  Sistem lebih bersifat penyajian data daripada analisis risiko.

- **User interface masih sederhana**  
  Bagi pengguna umum, navigasi layer dan pencarian data bisa membingungkan.

### Saran

- Peta zona rawan banjir (flood risk map) berbasis model hidrologi.
- Simulasi genangan banjir untuk memprediksi area terdampak.
- Integrasi laporan masyarakat (crowdsourcing data banjir).
- Informasi jalur evakuasi dan lokasi posko bantuan.
